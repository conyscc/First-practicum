GitHub основан на системе контроля версий Git, основной функцией является возможность командной разработки IT проектов с открытым исходным кодом.
Помимо открытого исходного кода, многие разработчики также размещают частные репозитории на платформе из-за удобства веб-сервиса. 
GitHub Issues - один из самых популярных трекеров для ошибок в мире. Он предоставляет возможность владельцам репозитория организовывать, помечать и связывать
вопросы с определенными этапами разработки.
На данной платформе так же можно подписаться на профиль интересующего разработчика или репозиторий, чья активность будет отображаться на панели инструментов.
Отличительной особенностью является система оценивания в звездах, с ее помощью можно также находить похожие проекты.
Вносить вклад в уже существуещие проекты, на которые у вас нет прав для внесения извемений путем отправки изменений, вы можете создав свое собственное ответвление проект (Fork)
Как только человек делает Pull Request, он рассматривается основными разработчиками проекта.
Кроме обратной связи и новых знакомств, платформа предоставляет некоторые функции по управлению проектами. 
Одна из таких функций - Projects
